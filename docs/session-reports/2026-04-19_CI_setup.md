## 7. Issues / Decisions

### Quyết định đã đưa (judgment calls)
- **Runner: `ubuntu-latest`, not `windows-latest`.** The task skeleton
  defaulted to windows-latest because "repo is Windows-primary", but that's
  a dev-environment property, not a test-suite property. The tests mock
  Playwright, never spawn Chrome, never hit hardcoded Windows paths, and
  use `tempfile.TemporaryDirectory()` everywhere. Ubuntu runners are ~2× faster
  and ~5× cheaper per minute — cost-of-inaction if we're wrong is tiny (flip
  one line back). Left a note in the report for future-us.
- **Single Python version (3.13) rather than a matrix across 3.11/3.12/3.13.**
  The project declares 3.13 is the prod runtime; testing older versions would
  create noise without protecting anyone. If multi-version support ever lands
  in SPEC, expand the matrix then.
- **`cache: pip` on setup-python.** Zero-config, uses hashes of
  `requirements*.txt` to key the cache. Saves ~20s per run after the first.
- **Coverage gate deferred.** Brief explicitly requested this. Baseline is
  ~30 %, R-TEST-4 target is 70 %. Adding `--cov-fail-under=70` today would
  fail every PR. Will land alongside Phase B coverage boost.
- **`pip install --upgrade pip` as the first step of install.** Cheap insurance
  against the occasional "this wheel needs pip≥24" surprise on fresh runners.
