## 7. Issues / Decisions

### Vấn đề phát sinh

**[Primary] Flow homepage locator is locale-hardcoded.** On `ngoctuandt20` the Flow UI renders Vietnamese content despite the engine appending `?locale=en` (log: `flow.operations.generate: On Flow homepage: https://labs.google/fx/tools/flow, locale=en`). `flow/operations/generate.py:125` raises:

```
RuntimeError: Failed to find '+ New project' button on Flow homepage
```

Evidence the account is **logged in and healthy** (not a credential loop):
- Page text preview (from worker log, mojibake is a secondary stdout-encoding issue — see below):
  `Flow / tv / Flow TV / help_outlined / Trung tâm trợ giúp về Flow / … / ULTRA / Apr 16, 08:49 PM / edit / Chỉnh sửa dự án / delete / Xoá dự án / …`
- Shows account tier (`ULTRA`), existing projects with timestamps, and Vietnamese action buttons (`Chỉnh sửa dự án` = "Edit project", `Xoá dự án` = "Delete project", `Trung tâm trợ giúp` = "Help Center").
- The "new project" entry point in VI is most likely `Dự án mới` or a plus-icon button — the English locator misses it.

This is the exact anti-pattern prohibited by **R-CODE-3 Locale-Independent** in `SPEC.md` and captured in `_TEMPLATE.md §6`.

**[Secondary] Windows stdout cp1252 cannot encode Vietnamese diacritics.** Log block at 2026-04-18 11:52:07 shows `UnicodeEncodeError: 'charmap' codec can't encode character '\u1ee3'` while trying to log the page-text preview. The error message itself was still emitted (via file handler), but stdout stream raised. Cosmetic on Windows; hides diagnostic text unless file log is read.

### Quyết định đã đưa

- **Halt chain at J1 per protocol** (`Step 2 rule 8`). Did not re-submit, did not skip to J2/J3. J2/J3 left `status=pending` — they will never claim because (a) `parent_job_id` points to a `failed` job and (b) no `project_url` was stored for inheritance. `GET /api/chains/{id}` correctly reports aggregate `status=failed`, `progress.completed=0/3` — B4 aggregation rule #1 ("any failed → failed") fired correctly. Incidental B4 validation.
- **Did not debug credentials.** Per FAILURE MODES, login-loop-class failures are BLOCKED; here the account is actually logged in so the blocker is different but the response is the same — don't modify code, document, commit.
- **Did not attempt to work around Chrome MCP to force-create a project manually.** The Tier 2 contract is "engine drives the browser"; bypassing would produce a result that doesn't describe engine behavior.
- **SPEC caveat worded as "not reached" rather than ✅/❌** — the B1/B11/B12 code paths literally did not execute under Tier 2. Falsely labeling ❌ would imply a regression we did not observe; falsely labeling ✅ would overstate evidence. "Not reached" is the honest state.

### Bug candidates phát hiện NHƯNG KHÔNG fix (out of scope)

- **[B-candidate-locale]** `flow/operations/generate.py:~125` — "+ New project" button locator is locale-hardcoded. Google Flow UI localizes the homepage even when `?locale=en` is in the URL (apparently driven by Google account locale preference, not URL). Fix direction: locate the button by stable DOM signal instead of visible text — candidates are (i) an SVG `<add>` / plus-icon child, (ii) anchor by click-target `<a href*="/project/new">`-like pattern if present, (iii) index-stable position on the nav (e.g. "first primary button in the top-left card"), or (iv) a multilingual regex union. Needs a DOM probe session on `ngoctuandt20` (similar to Tier1 probes for B11/B12). **Propose: add as B18 in SPEC.md §D.4 after supervisor review.** Severity: **P0 for any non-English account** — every `text-to-video` job on a VI (or other non-EN) Google account fails at the same point.
- **[B-candidate-stdout-encoding]** Windows stdout uses cp1252 by default (`python -m worker.main` on Windows 11). Any log record containing non-Latin characters raises `UnicodeEncodeError` in the StreamHandler. Fix direction: set `PYTHONIOENCODING=utf-8` in `worker/main.py` bootstrap, or `sys.stdout.reconfigure(encoding='utf-8', errors='replace')`, or configure the `StreamHandler` with `errors='replace'`. Severity: **P2** — cosmetic, file handler still captures full text, but masks diagnostics during live debugging.
